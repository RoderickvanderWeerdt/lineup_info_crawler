# Git / commit conventions

- Write commit messages that clearly describe what changed and why — no fixed prefix or tag format is required.
- Group changes into separate commits by logical concern; don't mix unrelated changes in one commit.
- Land larger changes as multiple small PRs that can be reviewed independently, rather than one large diff. Each PR description should state what changed and why, sized to be reviewable in one sitting.
- No special "AI-assisted" tag or label on PRs or commits — AI-assisted and human-authored changes follow the same norm: a clear description of what changed and why, not a marker of authorship.
